## TF-IDF R-EQA-style Refactor Check

### Goal

Verify that the new R-EQA-style entrypoint reproduces the original TF-IDF demo behavior.

The new entrypoint is:

```text
mini_eqa/baselines/tfidf_rag.py
```

### Commands

```

```

### Result

For `q1`, the top retrieved evidence is sofa-related:

```

frame_002 | A living room with a gray sofa and a small coffee table.
```

For `q2`, the top retrieved evidence is bathroom/mirror-related:

```

```

### Observation

The new modular entrypoint works correctly and preserves the behavior of the original one-file demo.

TF-IDF retrieves useful evidence when the question and caption share important words, such as `sofa` or `mirror`.

However, lower-ranked retrieved frames can still be irrelevant. For example, when `top_k=3`, some unrelated captions are included with low or zero scores.

### Limitation

The answer is still generated by `mock_runner.py`, so this version only validates retrieval and prompt construction. It does not evaluate real LLM answer quality.
